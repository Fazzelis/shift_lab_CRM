# CRM система для управления продавцами и транзакциями

## Стек технологий

| Категория | Технология | Назначение |
|-----------|-------|------------|
| **Язык** | Java 21 | Основной язык программирования |
| **Фреймворк** | Spring Boot 3.5.14 | Основной фреймворк приложения |
| **Базы данных** | PostgreSQL | Основная база данных |
| | H2 | Тестовая база данных (in-memory) |
| **ORM** | Spring Data JPA (Hibernate) | Работа с базой данных через JPA |
| **Сборка** | Gradle | Управление зависимостями и сборка |
| **Тестирование** | JUnit 5 | Юнит и интеграционное тестирование |
| | Mockito | Мокирование зависимостей |
| | Spring Boot Test | Интеграционное тестирование |
| | H2 | In-memory БД для тестов |
| **Утилиты** | Lombok | Генерация кода (геттеры, сеттеры, билдеры) |
| **Логирование** | SLF4J | Логирование приложения |

## Функциональность

### Продавцы (Sellers)
- Создание продавца
- Получение информации о продавце по ID
- Получение списка всех продавцов (с пагинацией)
- Получение списка удаленных продавцов (с пагинацией)
- Обновление информации о продавце
- Удаление продавца (с сохранением историчности данных)

### Транзакции (Transactions)
- Создание транзакции
- Получение списка всех транзакций (с пагинацией)
- Получение транзакции по ID
- Получение транзакций конкретного продавца (с пагинацией)

### Аналитика
- Самый продуктивный продавец за:
    - Произвольный период
    - День
    - Месяц
    - Квартал
    - Год
- Продавцы с суммой транзакций меньше указанной (за период, с пагинацией)
- Наилучший период для продавца (по количеству транзакций)

### Обработка ошибок

| HTTP статус     | Ситуация                             | Сообщение                                                                        |
|-----------------|--------------------------------------|----------------------------------------------------------------------------------|
| 400 Bad request | Передан неправильный формат значения | Неверный формат числа / кастомное сообщение                                      |
| 404 Not Found   | Продавец или транзакция не найдены   | Продавец не найден / транзакция не найдена / пустой массив / кастомное сообщение |

## Запуск проекта

### Требования
- Java 21
- PostgreSQL (локально или в Docker)

### 1. Клонирование репозитория

```bash
git clone https://github.com/Fazzelis/shift_lab_CRM.git
cd shift_lab_CRM
```

### 2. Сборка проекта и загрузка зависимостей
```bash
./gradlew clean build
```

### 3. Подготовка базы данных

Необходимо запустить PostgreSQL и создать базу данных под названием **crm_db**.

### 4. Настройка переменных сред окружения
В проекте используются следующие переменные:
- DB_URL = "jdbc:postgresql://адрес_бд:порт_бд/crm_db"
- DB_USERNAME = ваш_username
- DB_PASSWORD = ваш_password
- SERVER_PORT = ваш_порт_для_spring_boot (опционально, значение по умолчанию 8080)

Определить переменные можно следующим образом:
```bash
$env:DB_URL="jdbc:postgresql://адрес_бд:порт_бд/crm_db"
$env:DB_USERNAME="username"
$env:DB_PASSWORD="password"
$env:SERVER_PORT="8000"
```

### 5. Запуск приложения
```bash
./gradlew bootRun
```
Приложение запуститься на http://localhost:{указанный_порт|8080}

## Примеры использования API

В качестве примера использования API будет рассмотрена часть запросов.
Возьмем за базовый путь http://localhost:8080

### Продавцы

- Создание продавца
  - Post запрос на http://localhost:8080/seller
  - RequestBody:
```
{
  "name": "Test",
  "contactInfo": "test@gmail.com"
}
```

- Получение продавца по его id
  - Get запрос на http://localhost:8080/seller/{id}, где id является PathVariable
  - Пример строки запроса: http://localhost:8080/seller/1


- Получение всех продавцов
  - Get запрос на http://localhost:8080/seller/all
  - Query параметры: page (номер страницы), size (сколько элементов отображается на странице)
  - Пример строки запроса: http://localhost:8080/seller/all?page=2&size=10


- Получение всех продавцов
  - Get запрос на http://localhost:8080/seller/all
  - Query параметры: page (номер страницы), size (сколько элементов отображается на странице)


- Обновление данных продавца
  - Patch запрос на http://localhost:8080/seller/{id}, где id является PathVariable
  - Query параметры: page (номер страницы), size (сколько элементов отображается на странице)
  - RequestBody, где поля name и contactInfo являются необязательными:
```
{
  "name": "Test",
  "contactInfo": "test@gmail.com"
}
```

- Получение лучшего продавца за выбранный период
  - Get запрос на http://localhost:8080/seller/top-seller
  - Query параметры: startDate (в формате год-месяц-деньTчасы:минуты:секунды), endDate (в формате год-месяц-деньTчасы:минуты:секунды)
  - Пример строки запроса: http://localhost:8080/seller/all?startDate=2026-05-5T00:00:00&endDate=2026-05-12T23:59:59

### Транзакции

- Создание транзакции
  - Post запрос на http://localhost:8080/transaction
  - RequestBody:
```
{
  "sellerId": "1",
  "amount": "5462",
  "paymentType": "CARD"
}
```

- Получение всех транзакций
  - Get запрос на http://localhost:8080/transaction/all
  - Query параметры: page (номер страницы), size (сколько элементов отображается на странице)
  - Пример строки запроса: http://localhost:8080/transaction/all?page=2&size=10


- Получение транзакции по id
  - Get запрос на http://localhost:8080/transaction/{id}, где id является PathVariable
  - Пример строки запроса: http://localhost:8080/transaction/1


- Получение транзакции по id продавца
  - Get запрос на http://localhost:8080/transaction/seller-id/{id}, где id является PathVariable
  - Query параметры: page (номер страницы), size (сколько элементов отображается на странице)
  - Пример строки запроса: http://localhost:8080/transaction/seller-id/1?page=2&size=10
